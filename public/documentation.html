Documentation

This is an API that will give users access to various data about numerous movies including (but not limited to) genres, directors, and release dates. With this database, users can register with their own personal information and a create a list of their own favourite movies.

Movie Endpoints

Business Logic URL HTTP Method Request format Response format
Get the list of all movies /movies GET None JSON object holding data about all movies
Get the data about a single movie by title /movies/[Title] GET None JSON object holding data about single movie containing a title,description,genre,director,image URL.Example:
{
_id : ObjectId("5df2e16cf0c59ce6e6d51b90"),
Title : "Pulp Fiction",
Description: :"This film tells several stories of criminal Los Angeles.",
Genre: {
Name:"Drama",
Description: "Drama is a genre of narrative fiction (or semi-fiction) intended to be more serious than humorous in tone."
} ,
Director:
{
Name: Quentin Tarantino" ,
Bio: "Quentin Tarantino is an American filmmaker, actor, film programmer, and cinema owner.",
BirthYear: "1963
},
ImagePath: "pulpfiction.png" ,
Featured: false
}
Get the genre of a single movie by title /movies/genres/[Title] GET None A text message with the genre of the requested movie.Example:
Movie with the title: "Clueless" is a Comedy.

User Requests

Business Logic URL HTTP Method Request format Response format
Create new user /users/ POST A JSON object holding data about user to add with following structure:
{
Name: "Lights",
Password:"movie1",
Email: "actor@gmail.com",
Birthday:"1985-05-05",
Favorites: []
}
 JSON object holding data about user account that was just created. Example:
{
Name: "Lights",
Password:"movie1",
Email: "actor@gmail.com",
Birthday:"1985-05-05",
Favorites: []
}
Get/user profile /users/[Username] GET JSON object holding data about user account with the following structure:
{
Name: "Lights",
Password:"movie1",
Email: "actor@gmail.com",
Birthday:"1985-05-05",
Favorites: []
}
Update user information (username,,name, password, email, date of birth) /users/[Username] PUT A JSON object holding data about user to be updated with following structure:
{
Name: "Lights",
Password:"movie1",
Email: "actor@gmail.com",
Birthday:"1985-05-05",
Favorites: []
} 		A text message indicating whether the user info was successfully updated
Add a movie to user's list of favorit_movies /users/[Username]/Movies/[MovieID] POST A JSON object holding data about movie ID to be added to user's list of favourites.Example:
{
MovieID:"_id" : ObjectId("5df2d3bcf0c59ce6e6d51b89"
}
 JSON object holding data about updated user profile including the updated list of favorite movies.Example:
{
Name: "Camera",
Password:"movie2",
Email: "director@gmail.com",
Birthday:"1992-07-11",
Favorites: [ObjectId("5df2dbaff0c59ce6e6d51b8c"), ObjectId("5df2dd70f0c59ce6e6d51b8d"), ObjectId("5df2deebf0c59ce6e6d51b8e")]
}
Remove a movie from user's list of favorit_movies /users/[Username]/Movies/[MovieID] DELETE A JSON object holding data about MovieID to be removed from user's list of favourit_movies.Example:
{
MovieID:"_id" : ObjectId("5df2dd70f0c59ce6e6d51b8d")
}
 JSON object holding data about updated user profile including the updated list of favorite movies.Example:
{
Name: "Camera",
Password:"movie2",
Email: "director@gmail.com",
Birthday:"1992-07-11",
Favorites: [ObjectId("5df2dbaff0c59ce6e6d51b8c"), ObjectId("5df2deebf0c59ce6e6d51b8e")]
}
Delete user profile /users/[Username]/ DELETE A JSON object holding data about user to be deregistered with following structure:
{
Name: "Star",
Password:"movie5",
Email: "actress@gmail.com",
Birthday:"1988-10-10",
Favorites: []
}
 A text message indicating that the account with specific username was successfully deleted."
